## ЧАСТЬ 0. КОНТЕКСТ
═══════════════════════════════════════════

Я Антон, владелец **Rocksbox** — производство скалодромов под заказ (крупные коммерческие проекты, РФ). Юрлица: ООО Роксбокс + ИП Гладышев.
Автоматизирую цикл документов: заявка → КП → ТЗ → договор → производство → сдача.

**Ты не помнишь прошлые сессии — это нормально.** Контекст в этом файле + в форме `rocksbox_tz_form.html`. Веди этот файл аккуратно: в конце каждой сессии обновляй раздел «ГДЕ МЫ СЕЙЧАС». Это память для следующего раза (хоть для тебя, хоть для веб-Claude).

**Принципы (соблюдать строго):**
- Формулы считают — ИИ генерирует текст.
- Строим по одному звену. Не переусложнять. Edge case появился в реальном проекте → тогда добавляем поле. «Сад расходящихся тропок строим по мере того как по ним ходим.»
- Единый источник правды не превращать в помойку: у него чёткие входы/выходы.
- Антон не профессионал в логике систем, мысли иногда путаются — твоя задача структурировать и НЕ идти слепо на поводу, а честно говорить «так будет неудобно/запутаешься», как профессиональный архитектор систем.

## ЧАСТЬ 1. ЧТО МЫ СТРОИМ (архитектура)
═══════════════════════════════════════════

**КАРТОЧКА ПРОЕКТА = единый источник правды.** Одна форма-приложение (HTML). Заполняешь — из неё генерируются все документы. Каждый документ показывает только свою часть (заказчик не видит себестоимость, рабочие не видят цены).

```
Вход:  Excel-КП (площадь, суммы, состав, реквизиты) + ручное дозаполнение
   ▼
КАРТОЧКА ПРОЕКТА  (форма rocksbox_tz_form.html, данные в .json)
   ▼
Выходы (приложения к договору):
   - Приложение 1: ТЗ (описание объекта)
   - Приложение 2: Смета материалов и работ
   - Приложение 3: Условия выполнения работ (отношения, площадка)
   - Приложение 4: Календарный план (этапы: даты, привязка к оплате)
   + сам Договор (рамочный, рыба)
```

**Понятийно (Антону было важно):**
- Форма (`.html`) = ПРОГРАММА. Одна на всё. Не документ.
- Данные проекта (`.json`) = заполненная карточка проекта. Их много (по проекту).
- Открыл форму → открыл .json проекта → заполнил → сохранил → сгенерил документы.

## ЧАСТЬ 2. КЛЮЧЕВЫЕ АРХИТЕКТУРНЫЕ РЕШЕНИЯ
═══════════════════════════════════════════

### 2.1. Справочники типов (БИБЛИОТЕКА, копится по проектам)
Вместо повторного описания в каждом модуле — заводишь ТИП один раз, в модуле выбираешь из списка.

**Два уровня справочника:**

**А) Простые типы** (одна характеристика):
- Типы конструкций: полы / колонны / стены / потолок
- Типы панелей
- Типы матов

**Б) Составные типовые элементы** (целый мини-комплект):
- Kilterboard, Moonboard, Tensionboard, Campusboard — типовые тренажёры.
- У каждого зашито описание: рама, панели, двигатели, размер, стандарт разметки, требования к электрике.
- В проекте добавляешь модуль типа «Kilterboard» → подтягивается всё описание, дописываешь только крепление под объект.

### 2.2. НОСИТЕЛЬ vs ОТДЕЛКА (важно!)
Конструкция = носитель + отделка. Это РАЗНЫЕ поля:
- **Носитель** (пеноблок, бетон, кирпич, ж/б, сэндвич) — КОНЕЧНОЕ число, копится в библиотеку, выбор тегом/списком.
- **Отделка** (гипсокартон, плитка, штукатурка) — бесконечно разная, вводится РУКАМИ для проекта, в библиотеку НЕ копится.
- Если пол «ж/б 220» и «ж/б 220 + паркет» — это РАЗНЫЕ типы (отделка меняет тип).

### 2.3. Крепление — раздельно каркас и маты
К одному типу пола крепление КАРКАСА и крепление МАТОВ может отличаться. В модуле:
- крепление каркаса: к полу / к стенам / к колоннам / к потолку → выбор типа конструкции + способ крепления
- крепление матов: отдельные поля
Способы крепления (справочник): Нет / Анкерные болты / Химические анкеры / Сварка / Через деревянные пояса и лежень / Метрический крепёж

### 2.4. Механизм «применить ко всем»
У сквозных полей (Каркас, Покрытие, Толщина фанеры) — галочка «ко всем»: значение уходит в «Общее», в других модулях не спрашивается. Можно переопределить (исключение). Галочка СКРЫТА при одном модуле, появляется со второго.
(После введения справочников типов её роль уменьшилась, но оставить для сквозных вещей.)

### 2.5. Рост библиотеки — по проектам, через чистку (НЕ автонакопление)
- Внутри проекта: не хватило типа → кнопка «+ новый тип» прямо в форме (не уходя в код). Тип уезжает в .json проекта.
- ВАЖНО: справочник зашит в саму форму (в код). Автонакопления через localStorage для библиотеки НЕ делаем (плодит дубли).
- После проекта: Антон отдаёт .json веб-Claude → тот агрегирует новые типы, чистит дубли → пишет HANDOFF тебе → ты вшиваешь чистый справочник в форму.
- Дискретность роста = по проектам, через ревизию. НЕ после каждого микро-действия.

### 2.6. Автосохранение в браузере (СТРАХОВКА, не библиотека!)
Отдельная от библиотеки вещь. Форма автоматически сохраняет ТЕКУЩЕЕ состояние проекта в localStorage после каждого изменения. Закрыл вкладку/упал браузер → открыл, всё на месте. Без нажатий.
- Несколько проектов = несколько вкладок. Хранить КАЖДЫЙ проект под своим ключом (по названию проекта), чтобы вкладки не затирали друг друга.
- **Название вкладки браузера = название проекта** (document.title), чтобы Антон различал закреплённые вкладки.
- localStorage НЕ переживёт: чистку кэша, инкогнито, переустановку браузера. Поэтому кнопка «Сохранить в файл» (.json) остаётся — для бэкапа/переноса/передачи Claude. Жмётся по-крупному (конец проекта, бэкап), не после каждой буквы.
- Итог: автосохранение в браузер = ежесекундная страховка; JSON-файл = постоянный архив. Пара.

### 2.7. Форма — ОДНО ПОЛОТНО, не вкладки
Ввод — одно полотно сверху вниз (видно всё, не забудешь). Разделение на ТЗ/Условия/и т.д. — это про ВЫХОДНЫЕ документы, не про форму ввода. Вводишь в одном месте → раскладывается по приложениям на выходе.

**1. РЕКВИЗИТЫ И СТОРОНЫ** (шапка, для договора)
Заказчик (наименование/ФИО), тип (юрлицо/физлицо/ИП), контактное лицо, телефон, email, ИНН/КПП, ОГРН/ОГРНИП, паспортные данные (для физлиц), р/с, банк, БИК, корр.счёт, юр.адрес/адрес регистрации, подписант (ФИО, должность, основание). Адрес объекта, название проекта.

- 2.1 Типы конструкций: для каждого — носитель + отделка + способ крепления + рисунок-схема (скриншот разреза). Полы / Колонны / Стены / Потолок / Антресоли.

- 2.3 Типы матов
- 2.4 Составные типовые элементы (Kilterboard/Moonboard/Tensionboard/Campusboard)
- Коммуникации помещения: открытые / скрытые
- Несущая способность, ссылки на документацию здания (шифры проектов) — как в Тенгус
- Рисунки помещения (план Рис.1, разрезы)

**3. МОДУЛИ** [+ добавить модуль] — короткие, выбор из справочников
По каждому:
- Название (напр. Интро, Compwall, Kilterboard)
- Тип модуля: обычный / типовой элемент (если типовой — выбор из 2.4, подтягивается описание)
- Высота, Протяжённость/ширина основания (периметр для отдельностоящих)
- Геометрия — теги-чипы: вертикаль / слабые нависания / средние (до 15°) / сильные / нос-карниз / грот / нависание 30-40° / новички / обучение / соревнования / коммерческие трассы / детские соревнования (+ «свой»)
- Вылаз наверх (да/нет)
- Панель — выбор типа из 2.2
- Мат — выбор типа из 2.3
- Крепление каркаса: к полу / стенам / колоннам / потолку / антресоли (выбор типа конструкции + способ) + уточнение (напр. «к антресоли ОФП на высоте 3,6м через пояса и лежень»)
- Крепление матов (отдельно)
- Каркас / Покрытие / Толщина фанеры (с галочкой «ко всем»)
- Верхняя кромка (утолщённый край со скруглением / подрез к потолку / фанерная площадка)
- Комплектация (для тренажёров: каркас, панели, двигатели, без зацепок/подсветки)
- Особые требования (вырезы для окон; проём пожарного выхода с открыванием «от себя»; подвод электричества; «спиты не устанавливаются» и т.п.)
- Рисунки модуля

**4. ПАНЕЛИ (общие параметры, если едины)**
Материал/толщина фанеры, покрытие (полиуретановый лак/краска по дизайн-проекту, наносится на монтаже, резкий запах), сетка отверстий (М10, менее 25 шт/м², шаг 150-160мм; исключения — Kilter/Tension/Campus без отверстий), тип спитов (на саморезы, М10), люки доступа (крепятся винтами), фанерная юбка (вертикальная полоса 400мм вдоль основания для примыкания матов)

**5. МАТЫ (общие)**
Высота (напр. 300мм), покрытие (ткань ПВХ 900гр/м серая / армированный ковролin полиамид / войлочный ковролин на велкро), наполнитель послойно (верх ПВВ 45гр 50мм; низ EL/ST 22гр 250мм), тип чехлов, примыкания (к скалодрому через прижимные планки / к стенам / к проходам открытое через напольные планки)

**6. УСЛОВИЯ ВЫПОЛНЕНИЯ РАБОТ** (→ Приложение 3; данные про ОТНОШЕНИЯ)
- **Наряд-допуски:** работы на высоте (требуются/нет), огневые работы (требуются/нет)
- **Инструктаж:** периодичность (единоразово перед началом / иное)
- **Разгрузка:** тип места выгрузки (зона ТРЦ и т.п.), расстояние от парковки, наличие пандуса, допустимые размеры грузового транспорта (В/Ш/Д, напр. до 4м / 2,55м / 16,5м), путь разгрузки (грузовой лифт/лестница), размеры лифта (ДШВ, макс масса), пронос частей по лестнице (макс длина, напр. 4м), этаж монтажа, разгрузка через оконные проёмы (подъёмник, высота подъёма напр. 16,8м)
- **Готовность площадки (предоставляет Заказчик):** доступ свободный/не захламлено, отделка в местах примыкания демонтирована, коммуникации к демонтажу демонтированы, освещение (напр. 50 лк), электричество (220В 50Гц, мощность напр. 5кВт, отдельная линия с автоматом 16А — критично для Kilter/Tension), удалённость точек питания (не далее 50м), количество точек подключения (не менее 3)
- **Защита напольного покрытия** (кто предоставляет)
- **Место хранения инструмента на объекте**
- **Доступ/пропускной режим:** часы доступа (напр. 8:00–22:00), согласование въезда, взаимодействие с персоналом арендодателя
- **Выгородка рабочей зоны** (кто)
- **Вывоз строительного мусора** (вид, кто — обычно Подрядчик)
- **Уборка** (подмести/пропылесосить включая внутри скалодрома, кто — Подрядчик)
- **Смежные работы:** вид (монтаж антресолей/освещения/отделка пола), возможность параллельной работы (да/с ограничениями/нет), описание
- **Ограничения по шуму:** время работ с шумом до 100 дБ (напр. круглосуточно); время работ с шумом свыше 100 дБ; ограничения по дням недели
- **Температурный режим** для покраски/лака (если есть требования)
- **Складирование материалов** на объекте, охрана, ответственность за порчу
- **Приёмка скрытых работ** (крепления до закрытия — кто подписывает)

**ДОПУСК НА ПЛОЩАДКУ — подраздел (объединить с разгрузкой в смысловой блок «Допуск и логистика»):**
Поля-переключатели, чтобы форма помогала разграничить договорное/законодательное (подробная логика — в базе знаний `допуск_на_площадку.md`):
- Тип объекта: обычное здание / ОПО / завод (влияет на наряды, Ростехнадзор)
- Схема договора: один (ИП поставка+монтаж) / раздельно (ООО поставка + ИП монтаж)
- Сварка на объекте: да/нет → если да = ОГНЕВЫЕ РАБОТЫ обязательны (нельзя писать «огневых нет» при сварке). Альтернатива: вся сварка в цеху, на объекте только болтовой монтаж.

- Страхование на период работ: мы / заказчик / не предусмотрено
- Сертификат оформлен на: ООО / ИП — сверить с контрагентом по договору
- Класс горючести/КМ запрошен заказчиком: да/нет (панели = оборудование, не отделка; Г1 формально не на них, но могут запросить КМ по ФЗ-123)

«Не предусмотрено настоящим договором» — список того что прямо отклоняем (договорное, не законодательное):
страхование ГО, ППР для арендодателя (если письменно не требуют), СРО (не требуется для спортоборудования), проектная/исполнительная в объёме капстроя, огнезащитная обработка и лицензия.

ТЕРМИНОЛОГИЯ (зашить как подсказку): писать «поставка и монтаж спортивного оборудования», «монтажные/сборочные работы». ИЗБЕГАТЬ «строительно-монтажные работы», «реконструкция», «капитальное строительство» (тянут СРО и проектную документацию капстроя).

**7. ЭТАПЫ (→ Приложение 4 Календарный план + привязка к оплате)** [+ добавить этап]
По каждому: описание этапа, дата начала, дата конца, сумма по окончании (или % ), условие наступления (напр. «после согласования проекта»).

**8. ИТОГИ** (тянется/переносится из КП)
Общая площадь поверхности (напр. 1027 м²), итоговая сумма, режим НДС (без НДС / 5% ИП / 20%), срок действия цен (напр. 14 дней), курс евро для матов

**9. ГАРАНТИЯ** (раздельно по элементам!)
Каркас / Панели / Маты / Покрытие / тренажёры — у каждого свой гарантийный срок. (Решение Антона: единая гарантия невыгодна, покрытие истирается быстрее каркаса.)

## ЧАСТЬ 4. СТРУКТУРА ДОКУМЕНТОВ НА ВЫХОДЕ
═══════════════════════════════════════════

```
ДОГОВОР (рамочный, рыба — постоянный текст):
  Разделы тела: предмет, СУММА И ПОРЯДОК ОПЛАТЫ (остаётся в теле!),
    сроки и приёмка, обязательства сторон, ответственность, форс-мажор,
    изменение/расторжение, другие условия, гарантия, срок действия
  ├── Приложение 1: ТЗ (объект) — разделы формы 2,3,4,5
  ├── Приложение 2: Смета материалов и работ — из КП
  ├── Приложение 3: Условия выполнения работ — раздел формы 6
  └── Приложение 4: Календарный план — раздел формы 7
```

**Критерий разделения объект/отношения:**
